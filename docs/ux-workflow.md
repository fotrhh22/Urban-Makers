# 서울시 도시재생 체감지도 UX Workflow

## 1. 서비스 목적

서울시 도시재생활성화지역을 지도 기반으로 탐색하고, 구·법정동·활성화지역 단위로 사용자가 성과 지표와 사업 개요를 확인한 뒤 설문 참여 흐름으로 진입할 수 있도록 한다.

핵심 사용 시나리오:
- 서울시 내 자치구를 지도 또는 드롭다운에서 선택한다.
- 선택한 구에 속한 도시재생활성화지역 목록을 확인한다.
- 활성화지역을 선택하면 관련 법정동이 자동 선택된다.
- 상세보기에서 사업 개요를 확인한다.
- 설문조사 진입 버튼을 통해 의견 수렴 흐름으로 이동한다.
- 선택한 지역 기준의 지표 준비 상태를 확인한다.

## 2. 주요 화면 구조

### 2.1 Header

구성:
- 서비스명: `서울시 도시재생 체감지도`
- 탭: `대시보드`, `지표 설명`
- 데이터 업데이트 상태

역할:
- 서비스 정체성을 전달한다.
- 대시보드 화면과 지표 설명 화면을 전환한다.

### 2.2 Hero

문구:
- `서울시 도시재생 성과, 어떻게 생각하시나요?`
- `지역을 선택하고 다양한 지표를 통해 도시재생 성과를 함께 살펴보세요.`

역할:
- 사용자가 지도 탐색과 성과 확인, 의견 참여를 하게 될 것임을 안내한다.
- 현재 선택 기준의 활성화지역 개수를 요약한다.

### 2.3 Search Controls

구성:
- `구 선택` 드롭다운
- `법정동 선택` 복수 선택 탭

동작:
- 구 선택 시 선택 구가 변경된다.
- 구 선택 변경 시 기존 활성화지역 선택과 법정동 선택은 초기화된다.
- 법정동 탭은 복수 선택 가능하다.
- 법정동 탭 클릭 시 해당 동 선택 상태가 토글된다.

### 2.4 Map + Active Area List

구성:
- Google Maps 기반 서울 지도
- 서울시 자치구 경계 Polygon
- 선택 구 강조 표시
- 도시재생활성화지역 dot marker
- 우측 활성화지역 목록

동작:
- 지도에서 구를 클릭하면 해당 구가 선택된다.
- 선택된 구는 진한 fill/border로 강조된다.
- 선택된 구 이름은 흰색 배경 라벨 위에 진한 초록색으로 표시된다.
- 활성화지역 dot marker 클릭 시 해당 활성화지역이 선택된다.
- 우측 활성화지역 목록 클릭 시 해당 활성화지역이 선택된다.
- 같은 활성화지역을 다시 클릭하면 활성화지역 및 법정동 선택은 해제되고 구 선택은 유지된다.
- 여러 법정동에 걸친 활성화지역은 관련 법정동이 복수 선택된다.

### 2.5 Active Area Detail Modal

진입:
- 활성화지역 목록의 `상세보기` 버튼 클릭

표시 정보:
- 구분
- 유형
- 관할 자치구
- 법정동
- 선정연도
- 고시
- 상태

닫기:
- 우측 상단 `X` 버튼 클릭
- 모달 외부 배경 클릭

### 2.6 Survey CTA

노출 조건:
- 활성화지역이 선택된 경우에만 우측 목록 아래에 `설문조사하러가기` 버튼을 표시한다.

동작:
- 버튼 클릭 시 현재는 `준비중입니다` 알림을 표시한다.

### 2.7 Indicators

기본 상태:
- 활성화지역이 선택되지 않은 경우 `도시재생활성화지역 목록을 선택하면 지표 영역이 표시됩니다.` 안내를 표시한다.

선택 상태:
- 활성화지역 또는 법정동이 선택되면 지표 카드 영역을 표시한다.
- 현재 데이터는 초기화 상태이므로 각 지표는 `자료준비필요` 상태로 표시된다.

## 3. 핵심 사용자 흐름

### Flow A. 지도에서 구 선택

1. 사용자가 지도에서 특정 자치구를 클릭한다.
2. `구 선택` 값이 해당 구로 자동 변경된다.
3. 지도에서 해당 구가 강조된다.
4. 우측 목록은 해당 구의 도시재생활성화지역만 표시한다.
5. 법정동 선택과 활성화지역 선택은 초기화된다.

### Flow B. 드롭다운에서 구 선택

1. 사용자가 `구 선택` 드롭다운에서 구를 선택한다.
2. 지도 선택 상태가 해당 구로 변경된다.
3. 우측 활성화지역 목록이 선택 구 기준으로 필터링된다.
4. 기존 법정동/활성화지역 선택은 초기화된다.

### Flow C. 활성화지역 선택

1. 사용자가 우측 목록 또는 지도 marker에서 활성화지역을 선택한다.
2. 해당 활성화지역이 선택 상태가 된다.
3. 관련 법정동 탭이 자동 선택된다.
4. 여러 동에 걸친 사업이면 법정동이 복수 선택된다.
5. 하단 지표 영역이 열린다.
6. `설문조사하러가기` 버튼이 노출된다.

### Flow D. 활성화지역 선택 해제

1. 사용자가 이미 선택된 활성화지역을 다시 클릭한다.
2. 활성화지역 선택이 해제된다.
3. 자동 선택된 법정동도 해제된다.
4. 구 선택은 유지된다.
5. 지표 영역은 다시 안내 상태로 돌아간다.

### Flow E. 상세보기 확인

1. 사용자가 활성화지역 목록에서 `상세보기`를 클릭한다.
2. 사업 개요 모달이 열린다.
3. 사용자는 사업의 구분, 유형, 관할 구, 법정동, 선정연도, 고시, 상태를 확인한다.
4. `X` 버튼 또는 모달 외부 클릭으로 모달을 닫는다.

### Flow F. 설문조사 진입

1. 사용자가 활성화지역을 선택한다.
2. `설문조사하러가기` 버튼이 표시된다.
3. 버튼 클릭 시 `준비중입니다` 알림이 표시된다.

## 4. 상태 정의

### District Selection

기본값:
- `서울 전체`

변경 경로:
- 구 드롭다운 선택
- 지도 구 클릭
- 다중 자치구 활성화지역 선택 시 현재 구가 해당 지역에 포함되지 않으면 첫 번째 관할 구로 자동 이동

초기화 대상:
- 활성화지역 선택
- 상세보기 모달
- 법정동 선택

### Dong Selection

특성:
- 복수 선택 가능
- 수동 토글 가능
- 활성화지역 선택 시 자동 선택

초기화 조건:
- 구 변경
- 선택된 활성화지역 재클릭

### Active Area Selection

선택 경로:
- 지도 marker 클릭
- 우측 목록 클릭

선택 결과:
- 선택 활성화지역 강조
- 관련 법정동 자동 선택
- 지표 영역 표시
- 설문 CTA 표시

해제 경로:
- 동일 활성화지역 재클릭

### Detail Modal

열림:
- `상세보기` 클릭

닫힘:
- `X` 버튼
- backdrop 클릭
- 구 변경
- 선택 활성화지역 해제

## 5. 지도 UX 원칙

지도는 Google Maps를 배경으로 사용한다.

지도 로딩 설정:
- `useJsApiLoader`
- `language: "ko"`
- `region: "KR"`

Viewport:
- 지도 로드 또는 좌표 목록 업데이트 시 모든 자치구 경계와 활성화지역 marker가 화면 안에 들어오도록 `LatLngBounds`를 계산한다.
- 점이 1개뿐인 경우 `setCenter()`와 `setZoom(17)`을 사용한다.
- 일반 상황에서는 `fitBounds(bounds, 56)`을 사용한다.

시각 처리:
- 서울시 외곽은 dim 처리한다.
- 서울시 자치구 영역은 밝게 유지한다.
- 선택 구는 진한 초록색 경계와 fill로 강조한다.
- 선택 구 이름은 흰색 라벨 배경 위에 표시해 fill/border의 영향을 받지 않게 한다.

## 6. 현재 제한사항

- 설문조사 기능은 아직 실제 설문 페이지로 연결되지 않고 `준비중입니다` 알림만 표시한다.
- 지표 데이터는 초기화되어 있어 `자료준비필요` 상태로 표시된다.
- 활성화지역별 법정동 매핑은 현재 프론트엔드 상수 기반이다.
- 사업 상세 정보는 현재 제공된 기본 메타데이터 중심으로 표시된다.
